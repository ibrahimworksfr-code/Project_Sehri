# MADR-001: Upsampling via SMOTENC and L2 Penalized Logistic Regression

## Status

Accepted

## Date

2026-07-05

## Context

The initial exploratory data analysis of the $N=131$ Silver Layer revealed a severe class imbalance in the primary target variable, `high_value_night_intent` (123 negative instances to 7 positive instances). This imbalance was driven by sampling bias that predominantly captured price-sensitive students. 
Attempting to run standard predictive algorithms (like Logistic Regression or Random Forests) on a 5% positive class results in immediate failure; standard logistic models will trivially predict '0' for all instances, while tree-based algorithms will overfit to the 7 outliers. Furthermore, the extreme sparsity introduces the threat of quasi-complete data separation, where a specific geographic predictor could perfectly separate the outcome, driving Maximum Likelihood Estimation (MLE) parameters to infinity. 

## Decision

To salvage the predictive pipeline, we explicitly designed a Digital Twin architecture. We will deploy **SMOTENC (Synthetic Minority Over-sampling Technique for Nominal and Continuous data)** to upscale the minority class, generating a perfectly balanced 10,000-row matrix (5000:5000). 
Following this synthetic balancing, we will reject specialized quasi-complete separation algorithms (like Firth's Penalized Regression) and instead deploy a **Standard L2 (Ridge) Penalized Logistic Regression**, minimizing the objective function:

$$ J(\beta) = -\frac{1}{N} \sum_{i=1}^{N} [y_i \log(\hat{y}_i) + (1 - y_i) \log(1 - \hat{y}_i)] + \lambda \sum_{j=1}^{p} \beta_j^2 $$

## Consequences

* **Positive**: SMOTENC successfully handles the mixed data types (categorical text strings and continuous scores like `aov_score`), bypassing the interpolation fallacies associated with standard Euclidean SMOTE on nominal strings.
* **Positive**: The perfectly balanced 10,000-row matrix mathematically eliminates the quasi-complete separation problem, allowing standard MLE optimization to converge reliably.
* **Positive**: Deploying the L2 penalty prevents the logistic model from overfitting on the synthetic "lookalike" clones generated by SMOTENC, forcing the model to extract generalized patterns rather than memorizing noise.
* **Positive**: A logistic regression model provides explicit, interpretable feature coefficients (impact weights) which are required for stakeholder explainability in consulting environments, unlike black-box models.

## Rejected Alternatives

### 1. Firth's Penalized Logistic Regression
* **Why it was rejected**: Firth's regression introduces Jeffreys' invariant prior directly into the log-likelihood score function as a penalty term:

  $$ L^*(\beta) = L(\beta) + \frac{1}{2} \ln |I(\beta)| $$

  This mathematically anchors coefficients and prevents infinite standard errors during quasi-complete separation. However, once SMOTENC generated a perfectly balanced 5000:5000 dataset, the sparsity and separation threat was eliminated. Deploying Firth's on a massive, balanced matrix is computationally redundant and architecturally unnecessary. 

### 2. Standard SMOTE
* **Why it was rejected**: Standard SMOTE utilizes K-Nearest Neighbors to interpolate synthetic data points along Euclidean line segments. When applied to nominal categorical text (e.g., zone names), it hallucinates invalid geometric midpoints, completely corrupting the data structure.
